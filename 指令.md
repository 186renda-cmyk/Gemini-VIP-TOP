Gemini Balance 是一个开源的 API 代理和负载均衡工具，专门用于管理 Google 的 Gemini API。​​简单来说，它能帮你更高效、更稳定地使用 Gemini 的各项功能。
✨ 核心功能是什么？
如果你直接调用 Gemini API，可能会遇到 API Key 限额、请求失败、切换 Key 麻烦等问题。Gemini Balance 就是为了解决这些痛点而设计的，它主要有以下几个亮点：
		​​多 Key 负载均衡：​​ 你可以把多个 Gemini API Key 都放进去，它会自动轮换使用，避免单个 Key 被限速，从而提高整体的请求稳定性和并发量。
		​​双协议兼容：​​ 它不仅能处理 Gemini 自己的 API 格式，还能模拟 OpenAI 的 API 格式。这意味着你之前为 OpenAI 写的代码，几乎不做修改就能用上 Gemini 的能力。
		​​强大的功能支持：​​ 支持 Gemini 的各种高级功能，比如文本和图片的对话、生成新图片、编辑图片等。
		​​实时监控与管理：​​ 提供一个网页后台，可以实时查看每个 API Key 的状态和使用情况，方便你进行管理和监控。
		​​即改即生效：​​ 在网页后台修改配置后，服务会自动热更新，不需要重启，非常方便。
🛠️ 怎么用？适合谁？
这个工具本质上是一个需要自己搭建的服务，对于开发者来说，可以极大地简化工作流。

· Docker部署步骤：
1.克隆项目仓库：
git clone https://github.com/snailyp/gemini-balance.git
2.进入项目目录：
cd gemini-balance
3.复制并编辑
.env文件，配置API密钥、认证令牌等参数：
cp .env.example .env
vim .env
示例配置：
api_keys=("your-gemini-api-key-1","your-gemini-api-key-2")

auth_token="your-admin-token"
database_type=sqlite

docker run -d -p 8000:8000 --name gemini-balance --env-file .env -v $(pwd)/data:/app/data ghcr.io/snailyp/gemini-balance:latest

以下是关于 **Gemini Balance** 的使用教程，基于搜索结果中的关键信息整理而成。该工具是一个基于 Python FastAPI 开发的开源项目，用于管理 Google Gemini API 密钥并实现负载均衡功能，支持多密钥轮询、状态监控、图像生成等特性。

### **一、核心功能概述**
1. **多密钥负载均衡**  
   自动轮询多个 Gemini API 密钥，避免单一密钥失效导致服务中断。
2. **可视化监控面板**  
   实时查看密钥状态、API 调用统计、错误日志及健康状态。
3. **双协议兼容**  
   同时支持原生 Gemini API 和 OpenAI API 格式，无缝对接现有系统。
4. **智能故障处理**  
   自动重试失败请求，连续失败达阈值时禁用密钥。
5. **图像生成与图床上传**  
   集成图像生成功能及多平台图床支持。

---

### **二、安装与部署步骤**
#### **1. 环境准备**
- **系统要求**：Python 3.9+ 或 Docker 环境。
- **数据库**：MySQL 或 SQLite（生产环境推荐 MySQL）。

#### **2. 使用 Docker 部署（推荐）**
```bash
# 克隆项目
git clone https://github.com/snailyp/gemini-balance.git
cd gemini-balance

# 构建 Docker 镜像
docker build -t gemini-balance .

# 运行容器（映射端口 8000）
docker run -d -p 8000:8000 --env-file .env gemini-balance
```
> **注意**：需提前配置 `.env` 文件，包含 API 密钥、数据库连接等参数。

#### **3. 本地运行（非 Docker）**
```bash
# 安装依赖
pip install -r requirements.txt

# 启动应用
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

### **三、配置与使用指南**
#### **1. 添加 API 密钥**
- 通过监控面板 `/keys` 路径添加 Gemini API 密钥。
- 系统自动去重并启用有效密钥。

#### **2. 访问监控面板**
- 打开浏览器访问 `http://localhost:8000/keys_status`。
- **功能模块**：
  - **密钥状态**：区分有效/无效密钥，显示使用次数。
  - **调用统计**：图表展示 API 用量、成功率。
  - **错误日志**：记录并分析调用失败原因。

#### **3. 调用 API**
- **请求示例**（兼容 OpenAI 格式）：
  ```python
  import openai
  openai.api_key = "your-access-token"
  openai.api_base = "http://localhost:8000/v1"  # 指向 Gemini Balance 服务
  response = openai.ChatCompletion.create(model="gemini-pro", messages=[...])
  ```

### **四、高级配置**
1. **密钥轮换策略**  
   设置失败阈值（如连续失败 5 次禁用密钥），确保高可用性。
2. **流式输出优化**  
   启用 `stream=True` 参数，改善长文本生成体验。
3. **自定义域名**  
   通过 Cloudflare 添加 CNAME 记录，实现域名访问。

### **五、常见问题解决**
- **密钥失效**：在 `/keys_status` 页面检查状态，删除无效密钥。
- **DNS 问题**：在 Docker 命令中添加 `--dns 8.8.8.8` 指定 DNS。
- **模型列表不全**：设置 `URL_NORMALIZATION_ENABLED=false` 并调用 `/models` 同步。
